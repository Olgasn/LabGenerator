# Лабораторная работа 1: Разработка прототипа сетевого чата с шифрованием

## Цель работы
Разработать консольное клиент-серверное приложение для обмена текстовыми сообщениями с поддержкой сквозного шифрования. Студенты освоят базовые принципы сетевого программирования, работу с сокетами и реализацию криптографических алгоритмов.

## Постановка задачи
Создать два приложения: сервер, обрабатывающий подключения клиентов, и клиент, подключающийся к серверу. Обеспечить безопасную передачу сообщений между клиентами через сервер с использованием симметричного шифрования.

## Контракт ввода-вывода

### Сервер:
- **Вход**: Порт для прослушивания (например, 12345)
- **Выход**: 
  - Лог подключений/отключений клиентов
  - Пересылка зашифрованных сообщений между клиентами
  - Обработка ошибок сети

### Клиент:
- **Вход**: 
  - IP-адрес сервера
  - Порт сервера
  - Имя пользователя
- **Выход**:
  - Отправка сообщений на сервер
  - Отображение входящих сообщений
  - Статус подключения

## Требования к реализации

1. **Базовый функционал**:
   - Сервер поддерживает одновременную работу с 5+ клиентами (использовать потоки или асинхронность)
   - Клиент может отправлять и получать сообщения в реальном времени
   - Сообщения форматируются как: `[ВРЕМЯ] <ИМЯ>: ТЕКСТ`

2. **Шифрование**:
   - Реализовать алгоритм AES-256 для шифрования сообщений
   - Обмен ключами по алгоритму Диффи-Хеллмана
   - Каждое сообщение должно иметь уникальный вектор инициализации (IV)

3. **Обработка ошибок**:
   - Проверка корректности ввода
   - Обработка разрыва соединения
   - Валидация целостности сообщений

4. **Особые случаи**:
   - Повторное подключение с тем же именем
   - Длинные сообщения (>1024 символов)
   - Одновременная отправка сообщений несколькими клиентами

## Вариативность заданий

Студенты получают индивидуальные варианты, отличающиеся:

1. **Протоколом передачи**:
   - TCP (базовый вариант)
   - UDP (усложненный вариант)

2. **Алгоритмом шифрования**:
   - AES-256 (базовый)
   - ChaCha20 (доп. задание)
   - RSA для обмена ключами (усложненный вариант)

3. **Дополнительные функции**:
   - Поддержка файловых вложений
   - Голосовая передача (для продвинутых)
   - Групповые чаты

## Критерии оценки

1. **Базовый функционал (40%)**:
   - Корректная работа клиента и сервера
   - Устойчивость к сетевым ошибкам

2. **Безопасность (30%)**:
   - Корректная реализация шифрования
   - Защита от MITM-атак

3. **Качество кода (20%)**:

4. **Дополнительные функции (10%)**:
