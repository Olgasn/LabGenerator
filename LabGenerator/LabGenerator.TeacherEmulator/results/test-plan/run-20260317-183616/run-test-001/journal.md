# Лабораторная работа 1: Разработка прототипа модуля сетевого взаимодействия

## Цель работы
Разработать и реализовать прототип модуля для безопасной передачи данных между клиентом и сервером с использованием пользовательского протокола поверх UDP, включая механизмы контроля целостности и порядковой доставки.

## Постановка задачи
Создать два приложения (клиент и сервер), которые обмениваются структурированными сообщениями через ненадежный канал связи с имитацией потери пакетов. Реализовать базовые механизмы надежной передачи данных поверх UDP.

## Контракт ввода-вывода

### Входные данные:
- IP-адрес и порт сервера (для клиента)
- Порт прослушивания (для сервера)
- Текстовое сообщение для передачи (для клиента)
- Вероятность потери пакета (0-100%)

### Выходные данные:
- Лог операций на стороне сервера и клиента
- Статистика передачи (количество отправленных/полученных пакетов, количество повторных передач)
- Подтверждение доставки или сообщение об ошибке

## Требования к реализации

### Обязательные компоненты:
1. **Формат пакета:**
   - Идентификатор пакета (4 байта)
   - Флаги (1 байт): ACK, FIN, SYN
   - Порядковый номер (2 байта)
   - Длина данных (2 байта)
   - Контрольная сумма (4 байта)
   - Полезная нагрузка (до 512 байт)

2. **Функциональные требования:**
   - Установка соединения (трехэтапное рукопожатие)
   - Передача текстовых сообщений с подтверждением доставки
   - Механизм повторной передачи при таймауте (1 секунда)
   - Корректное завершение соединения
   - Валидация контрольной суммы

3. **Обработка ошибок:**
   - Обнаружение поврежденных пакетов
   - Обработка таймаутов
   - Восстановление при потере пакетов
   - Обработка дубликатов пакетов

### Варианты индивидуальных заданий:
1. **Механизм контроля переполнения:**
   - Вариант 1: Окно размером 1 пакет
   - Вариант 2: Скользящее окно размером 4 пакета

2. **Алгоритм расчета контрольной суммы:**
   - Вариант 1: XOR всех 4-байтовых слов
   - Вариант 2: Сумма всех байт по модулю 2^32

3. **Стратегия повторной передачи:**
   - Вариант 1: Фиксированный таймаут 1с
   - Вариант 2: Адаптивный таймаут (увеличивается при повторных передачах)

### Тестовые сценарии:
1. Передача сообщения без потерь
2. Передача с 30% вероятностью потери пакетов
3. Передача поврежденного пакета (изменение одного байта в полезной нагрузке)
4. Передача двух сообщений подряд
5. Тест на обработку дубликатов пакетов

### Ограничения:
- Язык программирования: Python 3.8+
- Использовать только стандартную библиотеку (кроме socket, struct, hashlib, random, time)
- Максимальный размер полезной нагрузки: 512 байт
- Таймаут ожидания подтверждения: 1 секунда
- Максимальное количество попыток повторной передачи: 5

### Критерии оценки:
1. Корректность реализации протокола (40%)
2. Обработка ошибок и edge cases (30%)
3. Качество кода и документация (20%)
4. Полнота тестового покрытия (10%)

### Отчетность:

2. Скриншоты работы программы для всех тестовых сценариев
3. Краткое описание архитектуры решения
4. Анализ работы программы при различных условиях

### Рекомендации:
