  - Средний размер пакета
  - Количество отфильтрованных пакетов

1. Поддержка протоколов:
   - Ethernet II
   - IPv4
   - TCP/UDP
   - ICMP

2. Функциональность:
   - Корректная обработка фрагментированных пакетов
   - Поддержка BPF-фильтров
   - Обработка ошибок (некорректные пакеты, отсутствие прав и т.д.)
   - Возможность досрочного завершения по сигналу (Ctrl+C)

3. Ограничения:
   - Использование только стандартных библиотек языка
   - Максимальный размер буфера: 65535 байт
   - Время обработки одного пакета: не более 1 мс

1. Дополнительный протокол для анализа (например, ARP, IPv6, DNS)
2. Дополнительный тип фильтрации (по содержимому пакета, по временному интервалу)
3. Дополнительная статистика (топ-источников трафика, график нагрузки)

### Особые случаи:

2. Пакеты с неизвестным протоколом
3. Переполнение буфера
4. Отсутствие прав доступа к сетевому интерфейсу
