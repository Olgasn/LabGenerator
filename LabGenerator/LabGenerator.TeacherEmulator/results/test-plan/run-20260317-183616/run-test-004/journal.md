# Лабораторная работа 1: Разработка прототипа сетевого модуля для обработки пакетов

## Цель работы
Разработать и реализовать прототип сетевого модуля, способного анализировать и обрабатывать сетевые пакеты с учетом различных протоколов передачи данных. Студенты получат практический опыт работы с сетевыми стеками, разбором пакетов и реализацией базовых сетевых операций.

## Постановка задачи
Необходимо создать программу на языке C/C++ или Python, которая:
1. Перехватывает сетевые пакеты в реальном времени
2. Анализирует заголовки различных сетевых протоколов
3. Выполняет фильтрацию пакетов по заданным критериям
4. Формирует статистику сетевого трафика

## Контракт ввода-вывода

### Входные данные:
- Сетевой интерфейс для мониторинга (например, eth0, wlan0)
- Конфигурационный файл в формате JSON с параметрами:
  - Порт(а) для мониторинга
  - Протоколы для анализа (TCP/UDP/ICMP)
  - Правила фильтрации
  - Параметры вывода статистики

### Выходные данные:
1. Лог-файл с детализированной информацией о перехваченных пакетах
2. Статистический отчет в формате:
   - Количество пакетов по протоколам
   - Средний размер пакета
   - Частота пакетов в секунду
   - Список IP-адресов с наибольшей активностью

## Требования к реализации

### Обязательные требования:
1. Поддержка как минимум трех протоколов транспортного уровня (TCP, UDP, ICMP)
2. Реализация системы фильтрации с поддержкой комбинированных условий
3. Корректная обработка фрагментированных пакетов
4. Защита от переполнения буфера
5. Многопоточная обработка пакетов

### Ограничения:
- Использование только стандартных библиотек языка
- Максимальный размер буфера для пакета: 65535 байт
- Время обработки одного пакета не должно превышать 1 мс
- Потребление памяти не более 100 МБ

### Граничные случаи:
1. Неполные или поврежденные пакеты
2. Пакеты с некорректными контрольными суммами
3. Одновременный входящий и исходящий трафик
4. Высокая сетевая нагрузка (более 1000 пакетов/сек)

## Вариативность заданий

### Параметры, варьируемые между студентами:
1. **Тип протокола прикладного уровня** (HTTP, DNS, DHCP и т.д.)
2. **Критерии фильтрации** (по IP-адресу, порту, флагам TCP и т.д.)
3. **Формат вывода статистики** (текстовый, CSV, JSON)
4. **Дополнительные функции** (например, обнаружение аномалий, генерация оповещений)

### Примеры индивидуальных заданий:
1. Реализовать детектор SYN-флуд атак
2. Добавить поддержку анализа DNS-запросов
3. Внедрить механизм геолокации IP-адресов
4. Реализовать визуализацию сетевого трафика

## Критерии оценки
1. Корректность работы (40%)
2. Полнота реализации требований (30%)
3. Качество кода и документации (20%)
4. Обработка граничных случаев (10%)
