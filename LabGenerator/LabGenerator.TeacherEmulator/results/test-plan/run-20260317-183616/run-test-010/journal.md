# Лабораторная работа 1: Разработка прототипа сетевого чата

## Цель работы
Разработать прототип консольного сетевого чата с использованием сокетов, реализующий базовые функции обмена сообщениями между клиентом и сервером. Работа направлена на освоение принципов сетевого взаимодействия, работы с сокетами и обработки сетевых исключений.

## Постановка задачи
Создать два приложения: сервер и клиент, которые взаимодействуют по протоколу TCP/IP. Сервер должен поддерживать одновременное подключение нескольких клиентов и обеспечивать пересылку сообщений между ними.

## Контракт ввода-вывода

### Сервер:
- **Входные данные:**
  - Порт для прослушивания (целое число)
  - Максимальное количество подключений (целое число)
- **Выходные данные:**
  - Логи подключений/отключений клиентов
  - Пересылаемые сообщения
  - Ошибки (если возникают)

### Клиент:
- **Входные данные:**
  - IP-адрес сервера
  - Порт сервера
  - Имя пользователя
- **Выходные данные:**
  - Входящие сообщения от других клиентов
  - Подтверждения о доставке сообщений
  - Ошибки подключения

## Требования к реализации

### Обязательные требования:
1. Серверная часть:
   - Поддержка одновременной работы с несколькими клиентами (использование потоков или асинхронного ввода-вывода)
   - Ведение списка активных подключений
   - Рассылка сообщений всем подключенным клиентам
   - Обработка отключений клиентов

2. Клиентская часть:
   - Возможность ввода сообщений с консоли
   - Отображение входящих сообщений в формате: `[Время] [Отправитель]: Сообщение`
   - Обработка разрыва соединения с сервером

3. Протокол обмена:
   - Сообщения передаются в кодировке UTF-8
   - Формат сообщения: `Имя_пользователя:Текст_сообщения`
   - Максимальная длина сообщения: 1024 байта

### Ограничения:
- Использовать только стандартные библиотеки языка программирования (без фреймворков)
- Поддерживать кодировку UTF-8
- Обрабатывать ошибки сетевого взаимодействия
- Обеспечить корректное завершение работы при получении сигнала прерывания (Ctrl+C)

### Граничные случаи:
1. Попытка подключения к несуществующему серверу
2. Отключение сервера во время работы клиента
3. Одновременная отправка сообщений от нескольких клиентов
4. Передача сообщения максимальной длины
5. Использование кириллических символов в именах и сообщениях

### Варианты индивидуальных заданий:
1. **Базовый уровень:**
   - Реализовать простой чат без истории сообщений
   - Поддержка только латинских символов

2. **Средний уровень:**
   - Добавить хранение истории сообщений (последние 100 сообщений)
   - Поддержка кириллицы
   - Команда /users для вывода списка активных пользователей

3. **Продвинутый уровень:**
   - Реализовать приватные сообщения (формат: @username сообщение)
   - Добавить шифрование сообщений
   - Реализовать аутентификацию пользователей

## Критерии оценки:

4. Соответствие требованиям варианта (10%)
