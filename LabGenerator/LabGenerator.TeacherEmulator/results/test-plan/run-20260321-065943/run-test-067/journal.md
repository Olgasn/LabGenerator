# Лабораторная работа 1: Разработка системы управления складом

## Цель работы
Разработать прототип системы управления складом с использованием принципов объектно-ориентированного программирования. Студенты научатся проектировать классы, реализовывать базовую бизнес-логику и обрабатывать исключительные ситуации.

## Постановка задачи
Необходимо создать консольное приложение для управления товарами на складе. Система должна позволять добавлять, удалять, искать и изменять товары, а также выводить различные отчеты.

## Контракт ввода-вывода

### Входные данные:
- Команды пользователя в текстовом формате
- Параметры команд (названия товаров, количества, цены и т.д.)
- Файл с начальными данными (опционально)

### Выходные данные:
- Текстовые сообщения о результате выполнения операций
- Отчеты в читаемом формате
- Сообщения об ошибках при некорректных операциях

## Требования к реализации

### Обязательные компоненты:
1. **Базовые классы:**
   - `Product` (товар) с полями: id, название, категория, цена, количество на складе
   - `Warehouse` (склад) с методами управления товарами
   - `WarehouseManager` для обработки команд пользователя

2. **Функциональность:**
   - Добавление нового товара
   - Удаление товара по ID
   - Поиск товаров по названию (частичное совпадение)
   - Изменение количества товара
   - Вывод списка всех товаров
   - Вывод товаров определенной категории
   - Подсчет общей стоимости всех товаров на складе

3. **Обработка ошибок:**
   - Проверка на отрицательное количество
   - Обработка дублирования ID
   - Валидация входных параметров
   - Обработка отсутствия товара

### Граничные случаи:
- Попытка удалить несуществующий товар
- Добавление товара с отрицательным количеством
- Поиск по пустой строке
- Изменение количества, приводящее к отрицательному значению
- Обработка некорректных числовых значений

### Вариативность между студентами:
1. **Дополнительные функции (на выбор):**
   - Сортировка товаров по цене/количеству/названию
   - Поиск по диапазону цен
   - История операций
   - Экспорт/импорт данных в файл
   - Резервное копирование данных

2. **Особенности реализации:**
   - Формат хранения данных (коллекции в памяти, файлы)
   - Алгоритмы поиска и сортировки
   - Детализация отчетов
   - Способ ввода/вывода (консоль, файлы)

## Критерии оценки
1. Корректность реализации бизнес-логики (40%)
2. Качество объектно-ориентированного дизайна (30%)
3. Обработка ошибок и граничных случаев (20%)
4. Читаемость кода и оформление (10%)

- Применять шаблоны проектирования, где уместно

- Документировать публичные методы
