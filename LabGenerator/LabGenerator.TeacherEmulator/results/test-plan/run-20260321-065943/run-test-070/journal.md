# Лабораторная работа 3: Оптимизация производительности и отказоустойчивости в ООП

## Цель работы
Разработка и оптимизация многопоточной системы обработки данных с механизмами отказоустойчивости и балансировки нагрузки. Студенты должны реализовать систему, способную эффективно обрабатывать параллельные запросы с гарантией целостности данных при сбоях.

## Постановка задачи
Спроектировать и реализовать систему "Распределённый кэш данных" (Distributed Data Cache), которая:
1. Обеспечивает хранение и быстрый доступ к данным
2. Поддерживает параллельную обработку запросов
3. Реализует механизмы отказоустойчивости
4. Включает систему мониторинга производительности

## Контракт ввода-вывода

### Входные данные:
- Конфигурационный файл в формате JSON, содержащий:
  - Параметры пула потоков (размер, политики)
  - Настройки кэша (размер, стратегия вытеснения)
  - Параметры репликации данных
  - Предельные значения для мониторинга (CPU, память, время отклика)

### Выходные данные:
1. Функционирующая система с REST API:
   - GET /cache/{key} - получение данных
   - PUT /cache - добавление/обновление данных
   - DELETE /cache/{key} - удаление данных
   - GET /metrics - метрики производительности
2. Логи операций в консоль и/или файл
3. Визуализация метрик производительности

## Требования к реализации

### Обязательные компоненты:
1. **Ядро кэша**:
   - Реализация на основе ConcurrentHashMap
   - Поддержка стратегий вытеснения (LRU, LFU, FIFO)
   - Время доступа O(1) для основных операций

2. **Многопоточная обработка**:
   - Пул потоков с настраиваемым размером
   - Очередь задач с приоритетами
   - Механизм graceful shutdown

3. **Отказоустойчивость**:
   - Сохранение состояния на диск при аварийном завершении
   - Механизм восстановления после сбоя

Создать приложение для управления научным экспериментом, включающее:
1. Планирование эксперимента с настраиваемыми параметрами
2. Сбор и хранение результатов
3. Статистический анализ данных
4. Визуализацию результатов

## Контракт ввода-вывода

### Входные данные:
- Конфигурационный файл в формате JSON с параметрами эксперимента
- Набор входных данных (CSV-файл) для обработки
- Параметры визуализации (тип графика, цветовая схема и т.д.)

### Выходные данные:
- Отчет о проведенном эксперименте (PDF/HTML)
- Графики и диаграммы результатов
- Статистический отчет (средние значения, стандартное отклонение, доверительные интервалы)

## Требования к реализации

### Обязательные компоненты:
1. **Класс `Experiment`**:
   - Хранит метаданные эксперимента (название, описание, дата)
   - Содержит список параметров и их допустимых значений
   - Реализует валидацию входных параметров

2. **Класс `DataCollector`**:
   - Загружает входные данные из CSV-файла
   - Обеспечивает предварительную обработку данных
   - Реализует кэширование результатов

3. **Класс `Analyzer`**:
   - Выполняет статистические расчеты
   - Реализует методы для проверки гипотез (t-тест, χ²-критерий)
   - Рассчитывает погрешности измерений и доверительные интервалы
   - Вычисляет средние значения и стандартное отклонение

4. **Класс `Visualizer`**:
   - Создает графики с использованием библиотеки Matplotlib
   - Поддерживает различные типы визуализации (линейные, столбчатые, круговые диаграммы)
   - Сохраняет графики в файлы

### Ограничения:
- Использовать только стандартную библиотеку Python и библиотеки NumPy, Pandas, Matplotlib
- Минимальный объем кода: 500 строк
- Поддержка как минимум 3 различных типов экспериментов
- Обработка ошибок для всех критических операций

### Граничные случаи:
- Пустые входные данные
- Некорректные значения параметров
- Отсутствие необходимых файлов
- Проблемы с правами доступа к файлам
