# Лабораторная работа 3: Оптимизация производительности и отказоустойчивости в ООП

## Цель работы
Разработать и реализовать систему кэширования для ресурсоемких вычислений с обеспечением отказоустойчивости при обработке больших объемов данных. Студенты освоят практические приемы повышения производительности и обеспечения устойчивости объектно-ориентированных систем.

## Постановка задачи
Создать систему кэширования результатов вычислений с поддержкой:
1. Многоуровневого кэширования (память + файловая система)
2. Механизма инвалидации устаревших данных
3. Обработки ошибок и восстановления после сбоев
4. Параллельного выполнения запросов

## Контракт ввода-вывода

  - Максимальный размер кэша в памяти
  - Время жизни записи в кэше
  - Путь для хранения файлового кэша
  - Политика замещения (LRU, LFU)

### Выходные данные:
- Результат вычислений для заданных параметров
- Статистика использования кэша:
  - Количество попаданий/промахов
  - Среднее время выполнения запроса
  - Загрузка памяти
  - Количество сбоев и восстановлений

   - Реализовать вычисляющую функцию (например, факториал, числа Фибоначчи, обработка строк)
   - Создать иерархию классов для кэширования
   - Реализовать два уровня кэширования (in-memory и файловый)

2. **Механизм кэширования:**
   - Автоматическое определение типа параметров для генерации ключа
   - Поддержка TTL (время жизни записей)
   - Реализация минимум двух политик замещения

3. **Обработка ошибок:**
