Разработать систему моделирования работы библиотеки с использованием бинарного поиска для эффективного поиска книг и анализа читательской активности.

## Постановка задачи
Создать программу, которая:
1. Моделирует каталог библиотеки с книгами
2. Реализует эффективный поиск книг по различным критериям
3. Анализирует статистику выдачи книг
4. Визуализирует результаты анализа

## Контракт ввода-вывода

### Входные данные:
- JSON-файл с описанием книг (название, автор, год издания, жанр, количество экземпляров)
- JSON-файл с историей выдачи книг

### Выходные данные:
- Отчет в формате Markdown с результатами анализа
- Графики популярности книг и жанров (гистограмма, линейный график, круговая диаграмма)
- Лог операций поиска и выдачи

## Требования к реализации

### Обязательные компоненты:
1. **Класс `Experiment`**:
   - Управление выполнением эксперимента
   - Координация работы всех компонентов
   - Формирование отчета

2. **Класс `DataGenerator`**:
   - Генерация тестовых данных книг
   - Создание истории выдачи книг
   - Поддержка различных распределений для генерации данных

3. **Класс `Library`**:
   - Управление каталогом книг
   - Обработка операций выдачи и возврата
   - Сортировка книг для бинарного поиска

4. **Класс `BookSearcher`**:
   - Реализация бинарного поиска по различным полям
   - Поддержка поиска по названию, автору, году издания

5. **Класс `LibraryAnalyzer`**:
   - Сбор статистики по популярности книг
   - Анализ читательского спроса по жанрам
   - Расчет метрик: среднее, медиана, дисперсия для времени поиска и выдачи

6. **Класс `LibraryVisualizer`**:
   - Построение рейтинга популярных книг (гистограмма)
   - Визуализация распределения книг по жанрам (круговая диаграмма)
   - График динамики выдачи по месяцам (линейный график)

### Требования к коду:
- Реализация бинарного поиска для эффективного поиска книг
- Обработка всех исключительных ситуаций
- Юнит-тесты с покрытием не менее 70% (покрытие проверяется инструментами тестирования)
- Подробная документация методов
- Соблюдение принципов SOLID

## Вариативная часть
- **Тип эксперимента**: Анализ эффективности поиска в библиотечном каталоге
- **Методы анализа**: Сравнение времени поиска при различных алгоритмах, регрессионный анализ
- **Форматы вывода**: Интерактивная веб-страница с возможностью фильтрации

## Ограничения
- Время выполнения эксперимента - не более 1 минуты
- Объем используемой памяти - не более 512 МБ
- Поддержка каталога до 10 000 книг
- Время поиска книги - не более 100 мс
- Поддержка пакетной обработки до 1000 запросов

## Граничные случаи
- Поиск несуществующей книги
- Попытка выдать отсутствующую книгу
- Поврежденные входные данные
- Одновременный доступ к одной книге
- Пустые входные данные
- Некорректные параметры эксперимента
- Отсутствие необходимых прав доступа к файловой системе
