# Лабораторная работа 1: Разработка прототипа сетевого модуля для обработки пакетов

## Цель работы
Разработать и реализовать прототип сетевого модуля, способного обрабатывать входящие пакеты данных согласно заданным правилам фильтрации и маршрутизации. Студенты получат практический опыт работы с сетевыми сокетами, парсингом заголовков пакетов и реализацией базовых сетевых функций.

## Постановка задачи
Необходимо создать консольное приложение, которое:
1. Перехватывает сетевой трафик на указанном сетевом интерфейсе
2. Анализирует заголовки пакетов (Ethernet, IP, TCP/UDP)
3. Применяет правила фильтрации из конфигурационного файла
4. Формирует логи обработки пакетов
5. (Дополнительно) Перенаправляет пакеты согласно таблице маршрутизации

## Контракт ввода-вывода

### Входные данные:
1. Аргументы командной строки:
   - `-i <interface>`: сетевой интерфейс для прослушивания (обязательный)
   - `-c <config_file>`: путь к файлу конфигурации (обязательный)
   - `-l <log_file>`: путь к файлу журнала (опционально, по умолчанию вывод в stdout)

2. Формат конфигурационного файла (JSON):
```json
{
  "filter_rules": [
    {
      "protocol": "tcp|udp|icmp|any",
      "source_ip": "CIDR",
      "destination_ip": "CIDR",
      "source_port": "number",
      "destination_port": "number",
      "action": "accept|drop|log"
    }
  ],
  "routing_table": [
    {
      "destination": "CIDR",
      "gateway": "IP",
      "interface": "string"
    }
  ]
}
```

### Выходные данные:
1. Журнал работы в формате:
```
[ВРЕМЯ] [УРОВЕНЬ] Протокол: PROTO, От: SRC_IP:PORT, Кому: DST_IP:PORT, Действие: ACTION
```
2. Статистика обработки пакетов по завершении работы (Ctrl+C)

## Требования к реализации

### Обязательные требования:
1. Поддержка протоколов: Ethernet, IPv4, TCP, UDP
2. Обработка минимум 3 правил фильтрации одновременно
3. Корректная обработка фрагментированных IP-пакетов
4. Валидация входных параметров и конфигурационного файла
5. Обработка сигналов завершения (SIGINT, SIGTERM)

### Варианты индивидуальных заданий:
Каждый студент получает уникальный набор параметров:

1. **Тип протокола** (TCP/UDP/ICMP)
2. **Правила фильтрации** (уникальный набор из 3-5 правил)
3. **Сценарий маршрутизации** (статическая маршрутизация для определенных подсетей)
4. **Дополнительные функции**:
   - Поддержка фрагментации/дефрагментации
   - NAT-трансляция
   - Поддержка VLAN-тегов
   - Ограничение скорости обработки пакетов

### Ограничения:
1. Запрещено использовать готовые библиотеки для фильтрации пакетов (например, libpcap)
2. Разрешено использовать только базовые сокеты (RAW sockets)
3. Код должен компилироваться под Linux
4. Максимальный размер буфера: 64KB

### Граничные случаи для тестирования:
1. Пакеты с некорректной контрольной суммой
2. Пакеты с TTL=1
3. Фрагментированные пакеты
4. Широковещательные и multicast-адреса
5. Пустые или поврежденные конфигурационные файлы

### Критерии оценки:
1. Корректность обработки пакетов (40%)
2. Полнота реализации требований (30%)

4. Качество кода и документация (10%)

## Рекомендации по выполнению:
1. Начните с реализации парсинга Ethernet-фреймов
2. Постепенно добавляйте поддержку IP, затем TCP/UDP
3. Используйте Wireshark для отладки
4. Протестируйте работу с помощью утилиты `nc` (netcat)
5. Убедитесь в корректности освобождения ресурсов при завершении
