# Лабораторная работа 1: Разработка прототипа сетевого приложения для передачи файлов

## Цель работы
Разработать консольное клиент-серверное приложение для безопасной передачи файлов между компьютерами в локальной сети с использованием пользовательского протокола поверх TCP/IP. Приложение должно обеспечивать базовую аутентификацию и контроль целостности передаваемых данных.

## Постановка задачи
Создать два модуля: клиент и сервер, взаимодействующие по сети. Сервер должен ожидать подключений на заданном порту, а клиент - подключаться к серверу и передавать файлы. Реализовать простой протокол обмена, включающий этапы аутентификации, передачи метаданных файла и собственно данных.

## Контракт ввода-вывода

### Сервер:
- Запуск: `server <port> <storage_directory>`
- Выход: 
  - При успешном запуске: "Server started on port <port>"
  - При ошибке: соответствующее сообщение об ошибке

### Клиент:
- Запуск: `client <server_ip> <port> <username> <password> <file_path>`
- Выход:
  - При успешной передаче: "File <filename> successfully transferred"
  - При ошибке: соответствующее сообщение об ошибке

## Требования к реализации

### Обязательные требования:
1. **Протокол обмена**:
   - Использовать TCP-сокеты
   - Формат сообщений: заголовок (4 байта - длина сообщения) + тело
   - Кодировка текста: UTF-8

2. **Этапы работы**:
   - Установка соединения
   - Аутентификация (простая, логин/пароль)
   - Передача метаданных (имя файла, размер)
   - Передача содержимого файла
   - Подтверждение приема
   - Закрытие соединения

3. **Обработка ошибок**:
   - Неверные учетные данные
   - Файл не найден
   - Ошибки сети
   - Недостаточно места на диске
   - Нарушение целостности данных

### Вариации между студентами:
1. **Алгоритм проверки целостности** (на выбор):
   - Контрольная сумма (CRC32)
   - Хеш (MD5, SHA-1)
   - Простая контрольная сумма

2. **Механизм аутентификации** (на выбор):
   - Простая пара логин/пароль
   - Хеширование пароля с солью
   - Использование токенов

3. **Дополнительные функции** (на выбор 1 из):
   - Прогресс-бар передачи
   - Возобновление прерванной передачи
   - Шифрование передаваемых данных

## Ограничения
- Язык программирования: Python 3.8+
- Использование сторонних библиотек: только стандартная библиотека Python
- Максимальный размер передаваемого файла: 100 МБ
- Таймаут соединения: 30 секунд

## Критерии оценки
1. Корректность работы (40%)
2. Обработка ошибок (25%)
3. Чистота кода и документация (20%)
4. Реализация дополнительных функций (15%)

4. Попытка подключения с неверными учетными данными
