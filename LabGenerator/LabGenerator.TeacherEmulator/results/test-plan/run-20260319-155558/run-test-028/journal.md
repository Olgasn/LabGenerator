# Лабораторная работа 1: Разработка прототипа сетевого сканера портов

## Цель работы
Разработать прототип сетевого сканера портов, способного определять состояние портов (открыт/закрыт) на заданном IP-адресе или доменном имени. Студенты получат практический опыт работы с сокетами, многопоточностью и сетевыми протоколами.

## Постановка задачи
Необходимо создать консольное приложение на языке Python, которое будет сканировать указанные порты на целевом хосте и выводить их состояние. Приложение должно поддерживать различные режимы сканирования и обработку ошибок.

## Контракт ввода/вывода

### Входные данные:
- Целевой хост (IP-адрес или доменное имя)
- Диапазон портов (начальный и конечный порт)
- Количество потоков (опционально, по умолчанию 10)
- Таймаут соединения в секундах (опционально, по умолчанию 1.0)

### Выходные данные:
- Таблица с результатами сканирования:
  - Номер порта
  - Состояние (OPEN/CLOSED/FILTERED)
  - Протокол (TCP/UDP)
  - Время отклика (мс)
- Статистика сканирования:
  - Общее количество проверенных портов
  - Количество открытых портов
  - Время выполнения сканирования

## Требования к реализации

### Обязательные требования:
1. Поддержка сканирования TCP-портов
2. Многопоточная реализация для ускорения процесса сканирования
3. Обработка следующих ошибок:
   - Неверный формат IP-адреса
   - Недоступность хоста
   - Некорректный диапазон портов
4. Возможность указания пользовательского таймаута
5. Вывод результатов в читаемом табличном формате

### Дополнительные возможности (на выбор студента):
1. Поддержка UDP-сканирования
2. Определение сервиса, работающего на открытом порту
3. Сохранение результатов в файл (JSON/CSV)
4. Графический интерфейс (Tkinter/PyQt)

### Ограничения:
- Запрещено использовать готовые библиотеки для сканирования портов (например, python-nmap)
- Максимальное количество потоков - 100
- Минимальный таймаут - 0.1 секунды

### Вариации между студентами:
1. **Алгоритм сканирования:**
   - Вариант 1: TCP Connect Scan
   - Вариант 2: TCP SYN Scan (полуоткрытое сканирование)
   - Вариант 3: TCP FIN Scan

2. **Дополнительные функции:**
   - Определение операционной системы целевого хоста
   - Проверка на наличие фаервола
   - Поддержка IPv6

3. **Способ вывода:**
   - Цветной вывод в консоль
   - Генерация отчета в формате HTML
   - Визуализация открытых портов

## Критерии оценки
1. Корректность работы (40%)
