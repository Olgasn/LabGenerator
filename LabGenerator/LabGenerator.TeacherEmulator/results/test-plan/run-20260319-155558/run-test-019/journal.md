# Лабораторная работа 1: Разработка прототипа сетевого приложения для передачи файлов

## Цель работы
Разработать консольное клиент-серверное приложение для безопасной передачи файлов по сети с использованием пользовательского протокола поверх TCP, обеспечивающего базовую аутентификацию и контроль целостности данных.

## Постановка задачи
Создать два приложения: клиент и сервер, взаимодействующие по заданному протоколу. Сервер должен аутентифицировать клиента, принимать файлы, проверять их целостность и сохранять в указанную директорию. Клиент должен подключаться к серверу, проходить аутентификацию и передавать файлы.

## Контракт ввода-вывода

### Сервер:
- **Входные параметры:**
  - Порт прослушивания (целое число, 1024-65535)
  - Путь к директории для сохранения файлов
  - Файл с учетными данными (логин:пароль:соль)
- **Выходные данные:**
  - Логи операций в консоль
  - Сохраненные файлы в указанной директории
  - Коды завершения: 0 - успех, 1-255 - ошибки

### Клиент:
- **Входные параметры:**
  - IP-адрес сервера
  - Порт сервера
  - Логин и пароль
  - Путь к передаваемому файлу
- **Выходные данные:**
  - Статус выполнения операции в консоль
  - Коды завершения: 0 - успех, 1-255 - ошибки

## Требования к реализации

### Обязательные требования:
1. **Протокол передачи:**
   - Использовать TCP-сокеты
   - Формат сообщений: [Длина заголовка (2 байта)][Заголовок JSON][Данные]
   - Поддерживаемые команды: AUTH, UPLOAD, BYE

2. **Аутентификация:**
   - Хранение паролей в виде хешей (SHA-256) с солью
   - Таймаут сессии 5 минут
   - Максимум 3 попытки ввода пароля

3. **Передача файлов:**
   - Максимальный размер файла: 100 МБ
   - Проверка целостности с помощью контрольной суммы (CRC32)
   - Поддержка возобновления передачи при обрыве соединения

4. **Обработка ошибок:**
   - Некорректные входные параметры
   - Ошибки сети
   - Ошибки аутентификации
   - Ошибки чтения/записи файлов

### Варианты индивидуальных заданий:
1. **Базовый уровень:**
   - Реализовать передачу одного файла за сессию
   - Простая аутентификация
   - Без поддержки возобновления передачи

2. **Продвинутый уровень:**
   - Множественная передача файлов
   - Шифрование данных (AES-256)
   - Прогресс-бар передачи

3. **Экспертный уровень:**
   - Поддержка докачки файлов
   - Многопоточная обработка клиентов
   - Генерация SSL-сертификатов

## Ограничения и граничные случаи

### Ограничения:
- Язык программирования: Python 3.8+
- Внешние библиотеки: только стандартная библиотека
- Максимальное количество одновременных подключений: 5 (для многопоточных вариантов)

### Граничные случаи:
1. Передача пустого файла
2. Попытка передачи несуществующего файла
3. Неверные учетные данные
4. Обрыв соединения во время передачи
5. Попытка перезаписи существующего файла
6. Недостаточно места на диске

## Критерии оценки
1. Корректность работы (40%)
2. Обработка ошибок (25%)
3. Соответствие протоколу (20%)
4. Качество кода (15%)
