# Лабораторная работа 1: Разработка прототипа сетевого модуля для обработки пакетов

## Цель работы
Разработать и реализовать прототип сетевого модуля, способного обрабатывать входящие сетевые пакеты, извлекать заголовки различных уровней сетевой модели OSI и выполнять базовый анализ сетевого трафика.

## Постановка задачи
Студентам необходимо создать программу на языке C/C++ или Python, которая:
1. Перехватывает сетевые пакеты в реальном времени
2. Анализирует заголовки различных уровней (Ethernet, IP, TCP/UDP)
3. Формирует статистику по сетевым соединениям
4. Реализует базовые функции фильтрации трафика

## Контракт ввода-вывода

### Входные данные:
- Сетевой интерфейс для захвата трафика
- Конфигурационный файл (JSON) с параметрами:
  - Период сбора статистики (в секундах)
  - Фильтры для отбора пакетов (по IP-адресам, портам, протоколам)
  - Порог для детектирования аномалий (количество пакетов/сек)

### Выходные данные:
1. Лог-файл в формате CSV со следующей структурой:
   - Временная метка
   - Источник (IP:порт)
   - Назначение (IP:порт)
   - Протокол
   - Размер пакета
   - Флаги TCP (если применимо)
2. Консольный вывод в реальном времени:
   - Текущая статистика по активным соединениям
   - Предупреждения о подозрительной активности
3. Итоговый отчет (по завершении работы):
   - Общее количество обработанных пакетов
   - Распределение по протоколам
   - Топ-5 IP-адресов по объему трафика
   - Обнаруженные аномалии (если есть)

## Требования к реализации

### Обязательные требования:
1. Поддержка как минимум трех протоколов: Ethernet, IPv4, TCP/UDP
2. Реализация кольцевого буфера для хранения последних N пакетов
3. Возможность динамического изменения фильтров без перезапуска программы
4. Обработка минимум 1000 пакетов в секунду без потерь
5. Корректная обработка фрагментированных IP-пакетов

### Ограничения:
- Использование только стандартных библиотек языка
- Максимальный размер буфера: 10 МБ
- Время инициализации программы: не более 2 секунд
- Потребление памяти: не более 50 МБ

### Граничные случаи:
1. Обработка пакетов с некорректной контрольной суммой
2. Корректное завершение работы при получении сигнала SIGINT
3. Обработка пакетов с нестандартными размерами заголовков
4. Работа при отсутствии сетевого интерфейса
5. Обработка пакетов с TTL=1

### Вариативность заданий:
Каждый студент получает уникальный вариант, который определяется по номеру в журнале:
1. **Протоколы**: Набор поддерживаемых протоколов (например, вариант 1: TCP, вариант 2: UDP, вариант 3: ICMP)
2. **Фильтры**: Различные комбинации фильтров (по MAC-адресам, диапазонам портов, типам сервисов)
3. **Аномалии**: Разные критерии детектирования подозрительной активности (флуд, сканирование портов, нестандартные флаги)
4. **Формат вывода**: Различные форматы лог-файлов (CSV, JSON, бинарный)
5. **Дополнительные функции**: Возможность сохранения дампа пакетов в формате pcap

1. Стратегией вытеснения кэша (LRU/LFU)
2. Параметрами балансировки нагрузки
3. Набором собираемых метрик
4. Алгоритмом проверки работоспособности нод
5. Частотой опроса метрик мониторинга

## Критерии оценки
1. Корректность работы системы (40%)
2. Качество кода и документации (25%)
3. Полнота реализации мониторинга (20%)
4. Эффективность балансировки нагрузки (15%)

## Рекомендуемые инструменты
- Язык программирования: Python/Go/Node.js
- Docker + Docker Compose
- Prometheus + Grafana
- Locust для нагрузочного тестирования

- Подготовка: 1 неделя
- Реализация: 2 недели
