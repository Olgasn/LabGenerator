# Лабораторная работа 1: Разработка прототипа сетевого приложения для передачи файлов

## Цель работы
Разработать консольное клиент-серверное приложение для надежной передачи файлов по сети с использованием протокола TCP/IP. Студенты освоят базовые принципы сетевого программирования, работу с сокетами и обработку ошибок при передаче данных.

## Постановка задачи
Необходимо реализовать два независимых приложения: клиент и сервер, которые обеспечивают передачу файлов между компьютерами в локальной сети. Приложение должно гарантировать целостность передаваемых данных и корректную обработку ошибок.

## Контракт ввода/вывода

### Серверная часть:
- **Входные параметры:**
  - Порт для прослушивания (целое число, по умолчанию 8080)
  - Путь для сохранения файлов (строка, по умолчанию текущая директория)
- **Выходные данные:**
  - Сохраненные файлы в указанной директории
  - Логи операций в консоль

### Клиентская часть:
- **Входные параметры:**
  - IP-адрес сервера (строка)
  - Порт сервера (целое число)
  - Путь к передаваемому файлу (строка)
- **Выходные данные:**
  - Статус передачи в консоль
  - Код возврата (0 - успех, ненулевое значение - ошибка)

## Требования к реализации

1. **Базовый функционал:**
   - Поддержка передачи файлов размером до 2 ГБ
   - Проверка доступности сервера перед передачей
   - Подтверждение получения файла сервером

2. **Обработка ошибок:**
   - Проверка существования исходного файла
   - Обработка разрыва соединения
   - Проверка доступности места на диске
   - Валидация входных параметров

3. **Протокол передачи:**
   - Передача имени файла и его размера перед передачей содержимого
   - Использование контрольной суммы для проверки целостности данных (CRC32)
   - Подтверждение получения каждого пакета данных

4. **Вариативность (для разных вариантов):**
   - Размер буфера передачи (1КБ, 4КБ, 16КБ, 64КБ)
   - Алгоритм контрольной суммы (CRC32, MD5, SHA-1)
   - Способ подтверждения получения (после каждого пакета/после всего файла)

5. **Особые случаи:**
   - Файл с таким именем уже существует на сервере
   - Недостаточно прав для записи в целевую директорию
   - Передача пустого файла
   - Передача файла с очень длинным именем (>255 символов)

## Критерии оценки

1. **Корректность работы (40%):**
   - Успешная передача файлов разного размера
   - Корректная обработка ошибок
   - Сохранение целостности данных

2. **Качество кода (30%):**
   - Читаемость и структурированность
   - Наличие комментариев
   - Обработка исключений

3. **Документация (20%):**
   - Описание выбранного варианта реализации
   - Инструкция по запуску
   - Описание протокола передачи

4. **Дополнительный функционал (10%):**
   - Прогресс-бар передачи
   - Возможность докачки файла
   - Поддержка передачи нескольких файлов за сеанс

## Рекомендации по выполнению

1. Начните с разработки простого эхо-сервера для отладки базового соединения
2. Реализуйте передачу метаданных (имя и размер файла) перед передачей содержимого
3. Добавьте проверку контрольной суммы после передачи
4. Протестируйте работу с файлами разного размера
5. Убедитесь в корректной обработке ошибок при разрыве соединения

## Варианты заданий

Каждому студенту будет назначен уникальный вариант, определяющий:
- Размер буфера передачи
- Алгоритм контрольной суммы

Пример варианта:
