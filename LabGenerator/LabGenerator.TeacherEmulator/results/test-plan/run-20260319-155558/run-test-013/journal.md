# Лабораторная работа 1: Разработка прототипа сетевого приложения для передачи файлов

## Цель работы
Разработать клиент-серверное приложение для передачи файлов с использованием сокетов TCP/IP, реализующее базовые механизмы сетевого взаимодействия, включая установление соединения, передачу данных и обработку ошибок.

## Постановка задачи
Создать два приложения: клиент и сервер, которые взаимодействуют по протоколу TCP/IP. Клиент должен иметь возможность подключаться к серверу, отправлять файлы и запрашивать список доступных файлов на сервере. Сервер должен обрабатывать входящие соединения, принимать файлы от клиентов и предоставлять список хранящихся файлов.

## Контракт входных/выходных данных

### Клиент:
**Входные данные:**
- IP-адрес сервера (строка)
- Порт сервера (целое число)
- Команда (строка: "UPLOAD", "DOWNLOAD" или "LIST")
- Путь к локальному файлу (для команд UPLOAD/DOWNLOAD)

**Выходные данные:**
- Для команды LIST: список файлов на сервере
- Для команды DOWNLOAD: запрошенный файл
- Статус выполнения операции (успех/ошибка)

### Сервер:
**Входные данные:**
- Порт для прослушивания (целое число)
- Путь к директории для хранения файлов (строка)

**Выходные данные:**
- Логи операций (подключения, загрузки, скачивания)
- Статус работы сервера

## Требования к реализации

### Общие требования:
1. Использовать язык программирования Python 3.8+
2. Реализовать протокол прикладного уровня поверх TCP
3. Обеспечить обработку ошибок (неверные команды, недоступность сервера, проблемы с файлами)
4. Реализовать логирование операций

### Требования к клиенту:
1. Поддержка трех команд:
   - UPLOAD <путь_к_файлу> - загрузка файла на сервер
   - DOWNLOAD <имя_файла> - скачивание файла с сервера
   - LIST - получение списка файлов на сервере
2. Проверка существования локальных файлов при загрузке
3. Обработка прерываний (Ctrl+C)

### Требования к серверу:
1. Поддержка одновременной работы с несколькими клиентами (многопоточность)
2. Проверка доступного места перед приемом файла
3. Защита от перезаписи существующих файлов
4. Ограничение максимального размера принимаемого файла (10 МБ)

### Ограничения:
1. Максимальный размер передаваемого файла: 10 МБ
2. Кодировка передаваемых данных: UTF-8
3. Таймаут соединения: 30 секунд
4. Максимальная длина имени файла: 255 символов

## Варианты заданий

### Вариант 1:
- Реализовать аутентификацию клиента по логину/паролю
- Добавить шифрование передаваемых данных с использованием AES
- Реализовать проверку целостности файлов с помощью контрольных сумм

### Вариант 2:
- Добавить поддержку докачки файлов (resume)
- Реализовать сжатие передаваемых данных с помощью gzip
- Добавить возможность установки прав доступа к файлам

### Вариант 3:
- Реализовать графический интерфейс для клиента
- Добавить поддержку передачи нескольких файлов за один сеанс
- Реализовать прогресс-бар для отображения хода передачи

## Критерии оценки
1. Корректность работы (40%)

4. Документация и комментарии (10%)
