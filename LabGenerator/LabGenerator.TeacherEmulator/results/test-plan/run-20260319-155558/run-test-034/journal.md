# Лабораторная работа 1: Разработка прототипа сетевого приложения для передачи файлов

## Цель работы
Разработать прототип клиент-серверного приложения для передачи файлов по сети с использованием сокетов, реализующий базовые механизмы сетевого взаимодействия и обработки ошибок.

## Постановка задачи
Создать два приложения: клиент и сервер, которые обеспечивают передачу файлов между компьютерами в локальной сети. Приложение должно гарантировать целостность передаваемых данных и обрабатывать основные сетевые ошибки.

## Контракт ввода-вывода

### Серверная часть:
- **Входные параметры**: 
  - Порт для прослушивания (целое число от 1024 до 65535)
- **Выходные данные**:
  - Сообщения о статусе работы сервера
  - Логи операций с файлами
  - Сохраненные файлы в указанной директории

### Клиентская часть:
- **Входные параметры**:
  - IP-адрес сервера
  - Порт сервера
  - Путь к отправляемому файлу
  - Имя файла для сохранения на сервере
- **Выходные данные**:
  - Статус передачи файла
  - Сообщения об ошибках (если возникают)

## Требования к реализации

### Обязательные требования:
1. **Базовый функционал**:
   - Сервер должен запускаться на указанном порту и ожидать подключений
   - Клиент должен подключаться к серверу и передавать файл
   - Поддержка передачи файлов размером до 100 МБ
   - Проверка целостности передаваемых данных (контрольная сумма)

2. **Обработка ошибок**:
   - Проверка доступности сервера
   - Обработка разрыва соединения
   - Проверка существования файла на клиенте
   - Проверка прав доступа к файлам

3. **Протокол взаимодействия**:
   - Использовать TCP-сокеты
   - Формат сообщений: первые 4 байта - размер сообщения, затем данные
   - Подтверждение получения каждого пакета

### Варианты индивидуальных заданий:
1. **Механизм возобновления передачи**:
   - Реализовать возможность докачки файла при разрыве соединения
   - Хранение информации о частично переданных файлах

2. **Шифрование данных**:
   - Реализовать шифрование передаваемых данных с использованием симметричного шифрования
   - Обмен ключами по алгоритму Диффи-Хеллмана

3. **Многофайловая передача**:
   - Поддержка передачи нескольких файлов за одно соединение
   - Архивация файлов перед передачей

4. **Прогресс передачи**:
   - Отображение прогресса передачи в реальном времени
   - Расчет и отображение скорости передачи

### Ограничения:
- Язык программирования: Python 3.8+
- Использование только стандартных библиотек (кроме вариантов с шифрованием)
- Максимальный размер буфера: 4096 байт
- Таймаут соединения: 30 секунд

### Тестовые сценарии:
1. Передача пустого файла
2. Передача большого файла (близкого к лимиту)
3. Попытка передачи несуществующего файла
4. Тестирование при разрыве соединения
5. Проверка целостности переданного файла

### Отчетность:
1. Исходный код приложения
2. Скриншоты работы программы
3. Описание архитектуры решения
4. Результаты тестирования
5. Выводы по работе
