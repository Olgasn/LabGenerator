Создать консольное приложение, способное:
1. Перехватывать сетевой трафик
2. Анализировать заголовки пакетов различных протоколов
3. Фильтровать пакеты по заданным критериям
4. Формировать статистику сетевой активности

### Входные данные:
- Имя сетевого интерфейса
- Фильтр пакетов (опционально)

2. Список IP-адресов с наибольшей активностью

4. Лог ошибок (при наличии)

## Требования к реализации

### Обязательные требования:
1. Поддержка протоколов:
   - Ethernet
   - IPv4/IPv6
   - TCP/UDP
   - ICMP

2. Функциональные модули:
   - Инициализация сетевого интерфейса
   - Парсинг заголовков пакетов
   - Система фильтрации (минимум 5 критериев)
   - Статистический анализ
   - Вывод результатов

3. Технические ограничения:
   - Язык программирования: Python 3.8+
   - Использование библиотеки Scapy
   - Кросс-платформенная работа (Windows/Linux)
   - Максимальный размер буфера: 1 МБ

### Вариации между студентами:
1. Уникальные критерии фильтрации:
   - Диапазон портов
   - Флаги TCP
   - Типы ICMP-сообщений
   - Подсети

2. Дополнительные функции:
   - Сохранение дампа пакетов в файл
   - Визуализация статистики
   - Обнаружение аномалий
   - Поддержка дополнительных протоколов

### Граничные случаи:
1. Обработка фрагментированных пакетов
2. Корректная работа при отсутствии сетевого интерфейса
3. Обработка поврежденных пакетов
4. Работа с большими объемами трафика

### Критерии оценки:
1. Корректность работы (40%)

3. Обработка ошибок (15%)
4. Качество кода (15%)
