## Цель
Разработать комплекс тестов для верификации корректности работы протоколов транспортного уровня (TCP/UDP) в различных сетевых условиях, включая аномальные ситуации.

## Задание
Создайте клиент-серверное приложение на Python 3.8 для тестирования базовых свойств сетевых протоколов. Реализуйте эмуляцию различных сетевых условий и проверьте устойчивость протокола к сетевым аномалиям.

### Требования к реализации:
1. Использовать модули: socket, threading, struct, time, json
2. Реализовать 5 обязательных категорий тестов:
   - Проверка целостности передаваемых данных
   - Тестирование обработки таймаутов соединения
   - Проверка обработки некорректных/битых пакетов
   - Измерение производительности при различных размерах передаваемых данных
   - Тестирование устойчивости к сетевым сбоям (потеря пакетов, задержки, джиттер)

### Граничные случаи:
- Передача пустого сообщения
- Передача данных максимального размера (1 МБ)
- Резкое обрывание соединения
- Многопоточная отправка данных (10 одновременных соединений)
- Симуляция 100% потери пакетов
- Тестирование при различных уровнях задержки (0-1000 мс)
- Тестирование при различном уровне джиттера (0-500 мс)

### Входные данные:
- IP-адрес и порт тестируемого сервера
- Набор тестовых сообщений различной длины (от 1 байта до 1 МБ)
- Параметры сетевых помех (задержка, джиттер, процент потери пакетов)
- Конфигурационный файл (config.json) с параметрами тестирования

### Выходные данные:
Отчет в формате JSON, содержащий:
- Статистику успешных/неудачных передач
- Измеренное время отклика при различных условиях
- Обнаруженные аномалии и дефекты протокола
- Рекомендации по настройке сетевых параметров
- Результаты тестирования при различных уровнях сетевых помех

## Ограничения
- Использовать только стандартные библиотеки Python
- Максимальный размер тестовых данных: 1 МБ
- Время выполнения тестов: не более 5 минут
- Поддержка IPv4 обязательно, IPv6 - опционально

## Критерии оценки
- Полнота тестового покрытия (30%)
- Корректность обработки граничных случаев (25%)
- Качество кода и документации (20%)
- Глубина анализа результатов тестирования (15%)
- Креативность тестовых сценариев (10%)
