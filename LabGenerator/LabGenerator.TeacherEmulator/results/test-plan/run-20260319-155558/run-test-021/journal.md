# Лабораторная работа 1: Разработка прототипа сетевого модуля для передачи файлов

## Цель работы
Разработать прототип клиент-серверного приложения для безопасной передачи файлов по сети с использованием пользовательского протокола поверх TCP, включая реализацию базовых механизмов контроля ошибок и целостности данных.

## Постановка задачи
Создать два приложения (клиент и сервер), взаимодействующие по сети с использованием протокола TCP. Приложение должно обеспечивать передачу файлов между клиентом и сервером с проверкой целостности данных и базовой обработкой ошибок.

## Контракт ввода-вывода

### Сервер:
- Принимает соединения на порту 8080
- Ожидает команды от клиента
- Поддерживает команды:
  - `UPLOAD <filename>` - принять файл
  - `DOWNLOAD <filename>` - отправить файл
  - `LIST` - показать список доступных файлов
  - `QUIT` - завершить сессию

### Клиент:
- Подключается к серверу по указанному IP-адресу и порту
- Отправляет команды серверу
- Обрабатывает ответы сервера
- Выводит результаты операций пользователю

## Требования к реализации

### Обязательные требования:
1. Реализовать протокол обмена сообщениями в формате:
   ```
   <длина_сообщения>:<команда>|<параметры>|<данные>
   ```
2. Для каждой передачи файла:
   - Рассчитывать контрольную сумму MD5
   - Передавать размер файла перед передачей содержимого
   - Подтверждать успешность приема
3. Обработка ошибок:
   - Неверные команды
   - Отсутствующие файлы
   - Ошибки передачи данных
   - Разрыв соединения

### Варианты индивидуальных заданий:
1. **Базовый уровень**:
   - Реализовать передачу текстовых файлов
   - Поддержка команд UPLOAD и DOWNLOAD
   - Простая проверка целостности по размеру файла

2. **Средний уровень**:
   - Добавить поддержку бинарных файлов
   - Реализовать команду LIST
   - Добавить проверку MD5

3. **Продвинутый уровень**:
   - Добавить сжатие передаваемых данных
   - Реализовать возобновление прерванной передачи
   - Добавить логирование операций

### Ограничения:
- Использовать только стандартные библиотеки языка
- Максимальный размер передаваемого файла - 100 МБ
- Таймаут соединения - 30 секунд
- Кодировка текстовых данных - UTF-8

### Тестовые сценарии:
1. Передача пустого файла
2. Передача большого файла (близкого к лимиту)
3. Попытка скачать несуществующий файл
4. Прерывание передачи
5. Передача файла с некорректными правами доступа

## Критерии оценки
1. Корректность работы (40%)

3. Качество кода (20%)
4. Соответствие протоколу (15%)
