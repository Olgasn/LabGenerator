# Лабораторная работа 1: Разработка прототипа сетевого приложения для передачи файлов

## Цель работы
Разработать консольное клиент-серверное приложение для передачи файлов по сети с использованием протокола TCP/IP, реализующее базовые механизмы сетевого взаимодействия и обработки ошибок.

## Постановка задачи
Создать два приложения: клиент и сервер, которые взаимодействуют между собой по сети. Сервер должен принимать входящие соединения, обрабатывать запросы на передачу файлов и сохранять полученные данные. Клиент должен уметь подключаться к серверу и передавать указанные файлы.

## Контракт ввода-вывода

### Сервер:
- **Входные параметры:**
  - Порт для прослушивания (целое число, 1024-65535)
- **Выходные данные:**
  - Лог операций в консоль
  - Сохраненные файлы в указанной директории

### Клиент:
- **Входные параметры:**
  - IP-адрес сервера
  - Порт сервера
  - Путь к передаваемому файлу
- **Выходные данные:**
  - Статус операции передачи в консоль

## Требования к реализации

### Обязательные требования:
1. Сервер должен:
   - Запускаться на указанном порту
   - Принимать несколько последовательных соединений
   - Принимать имя файла и его содержимое от клиента
   - Сохранять файл в поддиректорию `received_files`
   - Обрабатывать ошибки (недоступный порт, ошибки записи)

2. Клиент должен:
   - Устанавливать соединение с сервером
   - Передавать имя файла и его содержимое
   - Корректно обрабатывать разрыв соединения
   - Проверять существование локального файла

### Ограничения:
- Использовать только базовые сокеты (не использовать высокоуровневые библиотеки)
- Кодировка файлов: бинарный режим
- Максимальный размер файла: 100 МБ
- Таймаут соединения: 30 секунд

### Граничные случаи:
- Передача пустого файла
- Попытка передачи несуществующего файла
- Отключение клиента во время передачи
- Недостаточно места на диске
- Попытка перезаписи существующего файла

### Варианты индивидуальных заданий:
1. **Базовый уровень:**
   - Простая передача файлов без шифрования
   - Обработка основных ошибок

2. **Средний уровень:**
   - Добавить контрольную сумму (MD5) для проверки целостности
   - Реализовать прогресс-бар передачи

3. **Продвинутый уровень:**
   - Добавить базовое шифрование передаваемых данных (AES)
   - Реализовать аутентификацию клиента
