# Лабораторная работа 1: Разработка прототипа сетевого приложения

## Цель работы
Разработать прототип сетевого приложения, демонстрирующего работу ключевых сетевых протоколов и принципов взаимодействия в компьютерных сетях. Студенты получат практический опыт проектирования и реализации базовых сетевых компонентов.

## Постановка задачи
Требуется создать консольное сетевое приложение, способное устанавливать соединение между клиентом и сервером, обрабатывать входящие и исходящие сообщения, а также корректно обрабатывать возможные сетевые ошибки.

## Контракт ввода-вывода

### Серверная часть:
- Принимает входящие TCP-соединения на указанный порт
- Обрабатывает полученные от клиента сообщения
- Отправляет подтверждения о получении сообщений
- Ведет лог всех операций

### Клиентская часть:
- Устанавливает соединение с сервером по указанному IP-адресу и порту
- Отправляет текстовые сообщения на сервер
- Принимает и отображает ответы от сервера
- Корректно завершает соединение

## Требования к реализации

### Обязательные компоненты:
1. **Базовый функционал:**
   - Реализация многопоточного сервера, способного обрабатывать несколько клиентов одновременно
   - Механизм установки и разрыва соединения
   - Обработка стандартных сетевых ошибок (таймауты, разрыв соединения)

2. **Протокол обмена:**
   - Разработать простой текстовый протокол обмена сообщениями
   - Реализовать подтверждение получения сообщений
   - Обеспечить кодировку UTF-8 для текстовых данных

3. **Обработка исключительных ситуаций:**
   - Корректная обработка неверных входных данных
   - Защита от переполнения буфера
   - Обработка прерывания работы программы

### Ограничения:
- Использование языка программирования: Python, Java или C++
- Запрещается использование высокоуровневых фреймворков, скрывающих детали сетевого взаимодействия
- Максимальный размер сообщения: 4096 байт

### Граничные случаи для тестирования:
1. Отправка пустого сообщения
2. Одновременное подключение нескольких клиентов
3. Резкое отключение клиента без завершения сеанса
4. Передача сообщения максимально допустимого размера
5. Попытка подключения к несуществующему порту

### Требования к оформлению:
- Читаемый и документированный код
- Инструкция по сборке и запуску
- Примеры использования
- Тестовые сценарии

### Критерии оценки:
1. Корректность работы (50%)
2. Качество кода и документация (30%)
3. Обработка ошибок и граничных случаев (20%)
