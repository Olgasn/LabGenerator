# Лабораторная работа 1: Разработка сетевого протокола прикладного уровня

## Цель работы
Разработать и реализовать упрощенный протокол прикладного уровня для обмена сообщениями между клиентом и сервером, включая механизмы установки соединения, передачи данных и обработки ошибок.

## Постановка задачи
Студентам необходимо создать клиент-серверное приложение, реализующее собственный протокол поверх транспортного уровня (TCP). Протокол должен обеспечивать базовые функции обмена текстовыми сообщениями между несколькими клиентами через центральный сервер.

## Контракт ввода-вывода

### Входные данные:
- Адрес и порт сервера
- Имя пользователя (уникальный идентификатор клиента)
- Текстовые сообщения для отправки

### Выходные данные:
- Подтверждение подключения к серверу
- Входящие сообщения от других клиентов
- Статусные сообщения о событиях в системе

## Требования к реализации

### Обязательные компоненты:
1. **Серверная часть:**
   - Обработка множественных подключений
   - Маршрутизация сообщений между клиентами
   - Ведение списка активных пользователей
   - Обработка отключений клиентов

2. **Клиентская часть:**
   - Установка соединения с сервером
   - Отправка и получение сообщений
   - Отображение входящих сообщений
   - Графический интерфейс пользователя (GUI) или консольный интерфейс

### Протокол должен поддерживать:
- Формат сообщений с заголовком и телом
- Коды операций (установка соединения, отправка сообщения, отключение)
- Обработку ошибок передачи

### Ограничения:
- Использование языка программирования Python, Java или C++
- Работа через сокеты TCP
- Поддержка кодировки UTF-8
- Обработка не менее 10 одновременных подключений

### Граничные случаи:
- Потеря соединения с сервером
- Попытка подключения с уже занятым именем пользователя
- Передача пустого сообщения
- Одновременная отправка сообщений от нескольких клиентов
- Неверный формат сообщения

### Дополнительные требования:
- Документирование протокола
- Логирование основных событий
- Обработка исключительных ситуаций
- Тестирование на локальной машине и в сетевом окружении

### Критерии оценки:
1. Корректность работы протокола (40%)
2. Полнота реализации требований (30%)
3. Качество кода и документации (20%)
4. Обработка ошибок и граничных случаев (10%)
```
