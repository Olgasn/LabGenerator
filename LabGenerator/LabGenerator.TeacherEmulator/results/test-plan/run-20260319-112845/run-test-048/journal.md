- Обеспечивает мониторинг состояния соединений
- Обрабатывает исключительные ситуации

## Контракт ввода-вывода

### Входные данные:
- Конфигурационный файл (JSON) с параметрами:
  - Максимальное количество соединений (3-15)
  - Таймаут ожидания (до 30 секунд)
  - Путь к файлу базы данных
  - Интервал проверки активности (5 минут)

### Выходные данные:
- Пул соединений с методами:
  - `getConnection()` - получение соединения
  - `releaseConnection(connection)` - возврат соединения
  - `getStats()` - статистика использования
  - `validateConnection(connection)` - проверка валидности соединения

## Требования к реализации

### Обязательные требования:
1. Реализовать класс `SQLiteConnectionPool` как Singleton
2. Использовать шаблон Factory Method для создания соединений
3. Реализовать механизм автоматического восстановления соединений
4. Реализовать проверку валидности соединения перед выдачей
5. Обеспечить потокобезопасность операций
6. Реализовать таймауты ожидания
7. Добавить логирование критических операций

### Обработка исключений:
- `ConnectionTimeoutException` - при превышении времени ожидания
- `PoolLimitExceededException` - при превышении лимита соединений
- `ConnectionUnavailableException` - при ошибках подключения к БД

### Ограничения:
- Минимальный размер пула: 3 соединения
- Максимальный размер пула: 15 соединений
- Таймаут ожидания соединения: не более 30 секунд
- Время жизни соединения без активности: 5 минут
