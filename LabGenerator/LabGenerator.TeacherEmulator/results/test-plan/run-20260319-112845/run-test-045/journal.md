# Лабораторная работа 3: Реализация системы кэширования с LRU, LFU и FIFO стратегиями

## Цель работы
Разработать систему кэширования с поддержкой алгоритмов LRU, LFU и FIFO, обеспечивающую высокую производительность при различных паттернах доступа к данным.

## Задание
Реализовать систему кэширования, включающую:

1. **Базовый класс Cache** с методами:
   - `get(key)` - получение значения по ключу
   - `put(key, value)` - добавление элемента
   - `remove(key)` - удаление элемента
   - `clear()` - очистка кэша
   - `get_stats()` - получение статистики

2. **Реализация стратегии LRU (Least Recently Used)**
   - Использование хеш-таблицы и двусвязного списка
   - Перемещение элементов в начало при обращении
   - Удаление последнего элемента при переполнении

3. **Реализация стратегии LFU (Least Frequently Used)**
   - Поддержка частоты использования элементов
   - Эффективное обновление счетчиков обращений
   - Удаление наименее используемых элементов

4. **Реализация стратегии FIFO (First In First Out)**
   - Использование очереди для отслеживания порядка вставки
   - Удаление самого старого элемента при переполнении
   - Быстрая вставка в конец очереди

5. **Требования к реализации**
   - Полная потокобезопасность с использованием ReentrantLock и Condition
   - Механизм инвалидации по времени жизни элементов
   - Обработка граничных случаев (null-значения, несуществующие ключи)
   - Максимальный размер кэша: 1000 элементов
   - Время отклика < 50 мс
   - Визуализация состояния кэша

6. **Тестирование**
   - Сравнение hit-ratio для разных стратегий
   - Нагрузочное тестирование с многопоточным доступом
   - Анализ производительности при различных паттернах доступа
   - Проверка обработки граничных случаев

## Входные данные
- Конфигурация: размер кэша, выбранная стратегия, время жизни элементов
- Последовательность операций доступа к данным

## Выходные данные
- Статистика работы кэша (hit/miss ratio, среднее время доступа)
- Визуализация состояния кэша
- Логи операций с отметками времени
- Отчет об обработке граничных случаев
