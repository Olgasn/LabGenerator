## Требования к реализации

### Обязательные компоненты:
1. **Базовый функционал:**
   - Класс `CharacterCache` с generic-параметром для типа персонажа
   - Методы `put(characterId, character)`, `get(characterId)`, `remove(characterId)`
   - Автоматическая очистка неиспользуемых персонажей

2. **Стратегия вытеснения:**
   - FIFO (First In First Out)

3. **Потокобезопасность:**
   - Реализация с использованием `synchronized`
   - Обработка конкурентного доступа

4. **Обработка ошибок:**
   - Проверка на null и недопустимые аргументы
   - Обработка переполнения кэша
   - Логирование ошибок в файл

### Ограничения:
- Минимальный размер кэша: 10 персонажей
- Максимальный размер кэша: 1000 персонажей
- Время жизни персонажа в кэше: 1 минута

### Граничные случаи:
- Попытка добавления персонажа с существующим ID
- Обращение к несуществующему ID персонажа
- Параллельное обновление характеристик персонажа
- Исчерпание лимита памяти для кэша
