## Цель работы
Разработать и проанализировать механизмы повышения производительности и отказоустойчивости сетевого приложения с использованием современных протоколов и алгоритмов балансировки нагрузки.

## Постановка задачи
Студентам необходимо создать сетевое приложение клиент-серверной архитектуры с реализацией механизмов балансировки нагрузки и отказоустойчивости. Приложение должно демонстрировать устойчивость к сбоям и эффективно распределять нагрузку между несколькими серверами.

## Контракт входных/выходных данных
### Входные данные:
- Конфигурационный файл с параметрами:
  - Список серверов (IP-адреса и порты)
  - Параметры алгоритма балансировки нагрузки
  - Настройки таймаутов и повторов
- Входящие сетевые запросы от клиентов

### Выходные данные:
- Статистика работы системы:
  - Распределение запросов между серверами
  - Время отклика
  - Количество успешных/неудачных запросов
- Логи работы системы с указанием времени обработки и статуса операций

### Обязательные компоненты:
1. Реализовать базовый сервер, обрабатывающий HTTP-запросы
2. Разработать балансировщик нагрузки с поддержкой как минимум двух алгоритмов:
   - Round Robin
   - Наименьшее количество соединений (Least Connections)
3. Обеспечить механизм проверки работоспособности серверов (health check)
4. Реализовать обработку ошибок при недоступности серверов

### Требования к отказоустойчивости:
- Автоматическое исключение нерабочих серверов из пула
- Возможность планового вывода серверов на обслуживание
- Восстановление работы после временных сбоев сети

### Ограничения:
- Использование языка программирования Python или Go
- Применение асинхронного программирования для обработки запросов
- Запрет на использование готовых фреймворков для балансировки нагрузки

### Граничные случаи для обработки:
- Одновременный отказ всех серверов
- Резкое увеличение нагрузки
- Частичная потеря сетевого соединения
