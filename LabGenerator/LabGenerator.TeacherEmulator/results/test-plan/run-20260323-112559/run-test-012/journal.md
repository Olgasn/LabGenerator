public class TCPServer implements Runnable {
    private int port;
    private ServerSocket serverSocket;
    
    public TCPServer(int port) {
        this.port = port;
    }
    
    @Override
    public void run() {
        try (ServerSocket serverSocket = new ServerSocket(port)) {
            System.out.println("Сервер запущен на порту " + port);
            while (true) {
                Socket clientSocket = serverSocket.accept();
                new Thread(new ClientHandler(clientSocket)).start();
            }
        } catch (IOException e) {
            System.err.println("Ошибка сервера: " + e.getMessage());
        }
    }
}

public class TCPClient {

        socket = new Socket(ip, port);
        out = new PrintWriter(socket.getOutputStream(), true);
        in = new BufferedReader(new InputStreamReader(socket.getInputStream()));
        
        new Thread(() -> {
            try {
                String response;
                while ((response = in.readLine()) != null) {
                    System.out.println("Сервер: " + response);
                }
            } catch (IOException e) {

        }).start();

  - Ограничение частоты переключений
- Кеширование частых запросов (TTL-настройки)
- Горячее добавление/удаление серверов без перезапуска
- Расширенное логирование и мониторинг

- Время восстановления после сбоя: < 3 секунды
- Максимальное количество бэкендов: 50
- Максимальное количество соединений: 10000
- Использование памяти: < 1 ГБ

### Граничные случаи:
- Отказ всех бэкенд-серверов
- Резкий рост нагрузки (до 10000 RPS)
- Сетевые проблемы между балансировщиком и серверами
- Невалидные запросы
- Перегрузка отдельных узлов
