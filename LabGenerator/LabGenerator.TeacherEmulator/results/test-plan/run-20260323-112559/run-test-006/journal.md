## Постановка задачи
Создать клиент-серверное приложение, реализующее простой протокол обмена сообщениями. Протокол должен обеспечивать базовые функции мессенджера с возможностью регистрации пользователей, аутентификации и обмена сообщениями в реальном времени.

- Конфигурационные параметры (IP-адрес сервера, порт)

- Текстовые сообщения

- Статус выполнения операций
- Полученные сообщения
- Системные уведомления

   - Отправка и получение сообщений в реальном времени

### Протокол обмена:
- Использовать TCP для установления соединения
- Собственный текстовый протокол поверх TCP
- Формат сообщений: заголовок (команда) + тело (данные)
- Поддержка следующих команд: REGISTER, LOGIN, SEND, LIST, LOGOUT

### Обработка ошибок:
- Неверные команды
- Попытки дублирующей регистрации
- Неудачная аутентификация
- Потеря соединения
- Некорректный формат сообщений

### Граничные случаи:
- Одновременный вход одного пользователя с разных устройств
- Отправка сообщения несуществующему пользователю
- Длинные сообщения (более 1000 символов)
- Специальные символы в логине и пароле
- Резкое отключение клиента
