Разработать систему управления рисками для программного обеспечения, способную идентифицировать, оценивать и минимизировать потенциальные угрозы в работе приложения. Студенты научатся применять принципы объектно-ориентированного проектирования для создания устойчивых систем с механизмами защиты от сбоев.

## Постановка задачи
Спроектировать и реализовать модуль анализа рисков, который интегрируется в существующую программную систему. Модуль должен обеспечивать мониторинг критических операций, оценку их стабильности и автоматическое принятие решений при возникновении нештатных ситуаций.

## Контракт ввода-вывода
**Входные данные:**
- Поток событий системы (логи операций, метрики производительности)
- Конфигурационные параметры системы (пороговые значения, политики реагирования)
- Внешние сигналы о состоянии среды выполнения

**Выходные данные:**
- Отчеты о выявленных рисках с оценкой критичности
- Рекомендации по минимизации рисков
- Автоматически примененные защитные меры
- Статистика работы системы за период

## Требования к реализации

### Функциональные требования
1. Реализовать иерархию классов для представления различных типов рисков
2. Спроектировать систему сбора и агрегации метрик
3. Разработать механизм оценки критичности рисков
4. Реализовать стратегии обработки различных типов сбоев
5. Обеспечить ведение журнала событий системы
6. Реализовать многопоточную обработку событий

### Нефункциональные требования
1. Время реакции на критические события не более 100 мс
2. Поддержка многопоточного доступа к системе мониторинга
3. Возможность горячей замены стратегий обработки рисков
4. Минимальное влияние на производительность основной системы
5. Масштабируемость до 10 000 событий в секунду

### Обработка граничных случаев
1. Обработка ситуации отсутствия данных для анализа
2. Корректное восстановление после сбоев в подсистеме мониторинга
3. Устойчивость к некорректным входным данным
4. Обработка перегрузки системы событиями
5. Обработка тупиковых ситуаций в многопоточной среде

## Критерии оценки
1. **Полнота реализации (40%)**:
   - Реализованы все требуемые классы и интерфейсы
   - Корректная обработка всех типов рисков
   - Полнота покрытия граничных случаев
   - Реализация многопоточной обработки

2. **Качество кода (30%)**:
   - Соблюдение принципов SOLID
   - Правильное применение шаблонов проектирования
   - Читаемость и поддерживаемость кода
   - Эффективное использование многопоточности

3. **Тестирование (20%)**:
   - Наличие модульных тестов
   - Тестирование граничных случаев
   - Проверка производительности системы
   - Нагрузочное тестирование
   - Тестирование многопоточной безопасности

4. **Документация (10%)**:
   - Описание архитектуры решения
   - Руководство по использованию
   - Комментарии в коде
   - Диаграммы классов и последовательностей

## Рекомендуемые технологии
- Язык программирования: Java 11+ или C# .NET Core
- Система сборки: Maven/Gradle или MSBuild
- Фреймворки тестирования: JUnit/NUnit, JMeter/Gatling
- Система контроля версий: Git
- Инструменты мониторинга: Prometheus, Grafana
