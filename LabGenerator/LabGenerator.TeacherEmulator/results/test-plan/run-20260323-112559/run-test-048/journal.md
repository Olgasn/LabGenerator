Разработать систему обработки данных с применением хеширования для оптимизации производительности и обеспечения отказоустойчивости.

### Требования к реализации
1. **Архитектура системы**:
   - Реализовать модульную архитектуру с возможностью горизонтального масштабирования
   - Использовать паттерн "Наблюдатель" для мониторинга состояния системы

2. **Обработка данных с хешированием**:
   - Реализовать механизм кэширования на основе хеш-таблиц
   - Использовать хеш-функции для быстрого доступа к часто используемым данным
   - Обеспечить обработку коллизий методом цепочек

3. **Отказоустойчивость**:
   - Реализовать механизм контрольных точек (checkpoints) для восстановления состояния
   - Обеспечить репликацию критических данных
   - Реализовать стратегию повторных попыток при сбоях

4. **Параллельная обработка**:
   - Организовать пул потоков для параллельной обработки данных
   - Использовать механизм блокировок для синхронизации доступа к общим ресурсам

### Конкретные задачи
1. Реализовать класс `DataProcessor` с методами:
   - `process_data(data_chunk)` - обработка порции данных
   - `get_statistics()` - получение статистики производительности
   - `save_checkpoint()` - сохранение состояния системы

2. Разработать класс `HashCache` для кэширования данных:
   - Метод `put(key, value)` - добавление данных в кэш
   - Метод `get(key)` - получение данных из кэша
   - Метод `remove(key)` - удаление данных из кэша

3. Реализовать обработку ошибок:
   - При перегрузке системы
   - При нехватке памяти
   - При сбоях ввода-вывода

### Тестирование
1. Написать unit-тесты для проверки:
   - Корректности работы хеш-функции
   - Обработки коллизий
   - Восстановления после сбоев

2. Провести нагрузочное тестирование:
   - Обработка 1 ГБ данных
   - Измерение времени отклика при пиковых нагрузках
   - Проверка отказоустойчивости при искусственном создании сбоев

### Отчет
Подготовить отчет, содержащий:
1. Описание архитектуры системы
2. Результаты тестирования производительности
3. Анализ эффективности использования хеширования

## Требования

### Функциональные требования:
1. Реализовать распределенный бинарный поиск в отсортированных массивах данных
2. Обеспечить проверку целостности данных перед поиском

- Минимум 2 стратегии обработки ошибок (повторные попытки, отказоустойчивость)

### Граничные случаи:
- Пустой массив данных
- Массив с одним элементом
- Поиск отсутствующего элемента
- Повреждение данных (нарушение сортировки)
- Обработка дубликатов
- Сбои в работе узлов системы
